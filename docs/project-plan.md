# OpenPlanner 기말 프로젝트 계획

## 목표

중간과제의 핵심 주장인 "AI가 약속을 결정한다"를 실제 데모로 증명한다. 현재 MVP는 방 안의 모든 참여자가 같은 화면에서 가능한 시간을 확인하는 공용 시간 결정 흐름에 집중한다.

기말과제 안내문 기준으로 가장 중요한 목표는 "실제로 실행되는 AI 웹 서비스 프로토타입"이다. 따라서 기능 범위를 넓히기보다, 사용자가 입력하고 AI 의사결정 로직이 처리한 뒤 결과가 화면에 표시되는 한 흐름을 안정적으로 완성한다.

## MVP

1. 로그인 및 개인 캘린더
   - 입력: 사용자 이름, 거주 지역, 일정명, 날짜, 시작/종료 시간, 공개 여부
   - 처리: 로그인 후 개인 캘린더 화면 표시
   - 출력: 월간 달력에서 직접 입력한 본인 일정 확인

2. 약속 방 생성 및 초대코드 입장
   - 입력: 주최자가 모임 이름 입력, 참여자가 초대코드·이름·거주 지역 입력
   - 처리: 초대코드 생성, 참여자들이 코드로 입장, 참여자별 일정 추가
   - 출력: 참여자 목록과 공개/비공개 일정 표시

3. 주최자 기간 설정
   - 입력: 시작일, 종료일
   - 처리: 해당 기간 안에서 후보 날짜와 시간 탐색
   - 출력: AI 시간 선정 실행 가능 상태

4. AI 시간 결정
   - 입력: 참여자별 공개/비공개 일정, 선호 시간대
   - 처리: 가능한 시간 후보를 만들고 평일 18시 이후 조건, 일정 충돌, 이동 여유로 점수화
   - 출력: 1-3순위 시간 후보와 이유

5. 공용 결과 화면
   - 입력: AI가 계산한 가능 시간 후보
   - 처리: 전원 가능 시간과 충돌 시간을 시각적으로 구분
   - 출력: 모든 참여자가 같은 화면에서 보는 1순위 시간, 가능 시간 보드, 후보 순위

6. LLM 메뉴·장소 추천
   - 입력: 참여자별 먹고 싶은 메뉴, 피하고 싶은 메뉴, 선호 장소, 원하는 분위기
   - 처리: LLM이 선호를 취합하고 타협 가능한 메뉴/장소를 추천
   - 출력: 추천 메뉴, 추천 장소, 추천 이유, 타협안 설명

## 구현 전략

### input data

- 사용자 입력 참여자
- 사용자 입력 공개/비공개 일정
- 사용자 입력 메뉴/장소 선호
- 자동 생성 초대코드

### algorithmic module

- 시간 후보 점수화
- 평일 18시 이전 후보 제외
- 가능/불가능 시간 보드 생성
- 1순위 시간 강조
- privacy 일정은 상세명 대신 바쁨으로 처리
- 참여자 선호 입력 취합
- LLM API 호출 실패 시 로컬 fallback 추천

### LLM prompt

Node 서버가 `OPENAI_API_KEY`를 읽어 OpenAI Responses API를 호출한다. LLM은 참여자별 메뉴/장소 선호와 가능 시간 후보를 입력받고, 추천 메뉴·장소·이유를 JSON으로 반환한다. API 키가 없거나 호출이 실패하면 앱은 로컬 fallback 추천으로 계속 실행된다.

## 제외

- 실시간 캘린더 동기화
- 카카오톡 자동 전송
- 지도 API
- 날씨 기반 D-day 알림
- 약속 안내 메시지 생성

## 제출 기준 대응

- 실행 결과물: `index.html` 또는 `node server.js`로 실행 가능한 웹앱
- 핵심 기능: 사용자 입력 로그인, 개인 일정 입력, 방 생성, 초대코드 입장, 참여자 일정 입력, 기간 설정, AI 시간 선정, LLM 메뉴·장소 추천, 공용 결과 화면
- 설명문서: 서비스 소개, 주요 기능, AI 입출력, 실행 방법, 한계와 개선 방향을 PDF로 정리
- 시연 영상: 로그인부터 AI 시간 선정 결과 확인까지 3-5분 흐름으로 녹화

## 데모 흐름

1. 로그인 화면에서 이름과 거주 지역을 입력한다.
2. 개인 캘린더에 본인 일정을 직접 추가한다.
3. 주최자가 약속 방을 만들고 초대코드를 생성한다.
4. 다른 참여자들이 초대코드, 이름, 거주 지역을 입력해 입장한다.
5. 참여자별 일정을 직접 추가한다.
6. 참여자별 메뉴/장소 선호를 입력한다.
7. 주최자가 만날 기간을 지정한다.
8. `AI 시간 선정`을 누른다.
9. 모든 참여자의 일정표를 기반으로 전원 가능 시간 보드가 표시된다.
10. LLM이 참여자별 메뉴/장소 선호를 취합해 추천을 생성한다.
