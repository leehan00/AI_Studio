## 제출물

1. 실행 가능한 결과물
   - 소스 코드 전체
   - 실행 방법이 적힌 README
   - 핵심 기능이 처음부터 끝까지 동작해야 함

2. 설명문서 PDF
   - 서비스 소개
   - 주요 기능 설명
   - AI의 역할과 입출력
   - 실행 방법
   - 배운 점, 한계, 개선 방향

3. 실행 시연 동영상
   - 실행 과정
   - 핵심 기능 사용 장면
   - 입력, AI 처리, 결과 확인 흐름
   - 3-5분 내외 권장

## 평가 기준

| 평가 항목 | 비중 | OpenPlanner 대응 |
| --- | ---: | --- |
| 결과물이 실제로 실행되는가 | 35% | `index.html`로 실행 가능한 웹 프로토타입 |
| 핵심 기능과 AI 활용이 의도대로 동작하는가 | 30% | 참여자 캘린더 입력, AI 시간 선정, LLM 메뉴·장소 추천, 공용 결과 화면 |
| 설명문서가 결과물을 명확히 설명하는가 | 20% | PDF 설명문서 작성 필요 |
| 중간 기획을 의미 있게 고도화·구현했는가 | 15% | 중간과제의 의사결정 엔진을 방/캘린더/시간선정 흐름으로 구체화 |

## 현재 상태

충족:

- 웹에서 실행 가능
- 로그인 데모 흐름 있음
- 개인 캘린더 화면 있음
- 방 생성 및 초대코드 입장 흐름 있음
- 주최자 기간 설정 있음
- 참여자 일정 기반 가능 시간 계산 있음
- 공용 결과 화면 있음
- 참여자별 메뉴/장소 선호 입력 있음
- OpenAI API를 통한 LLM 추천 서버 있음
- API 키가 없어도 fallback 추천으로 실행 유지

보강 필요:

- AI 활용 설명 강화: 시간 계산은 규칙 기반, 메뉴·장소 추천은 LLM 기반으로 역할을 분리해 설명해야 함
- 설명문서 PDF 작성
- 시연 영상 대본 작성
- 깨끗한 환경에서 실행 확인

## 다음 우선순위

1. 현재 정적 앱을 제출 가능한 수준으로 안정화한다.
2. AI의 입력, 처리, 출력을 화면과 문서에서 더 명확히 보이게 한다.
3. 설명문서 초안을 작성한다.
4. 시연 영상 흐름을 3-5분 대본으로 정리한다.
5. 필요하면 LLM API 연결을 추가하되, 실행 안정성을 해치지 않는 범위에서만 진행한다.
