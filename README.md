1. **김지우**로 로그인 → 약속 방(이날 어때) 탭 → 방 만들기 → 초대코드 복사
2. (다른 브라우저/시크릿 창에서) **이서연·박준호·최유진**으로 로그인 → 초대코드 입장 → 선호 입력 → 준비 완료
3. 김지우가 만날 기간 지정 후 **AI 시간 선정** → 전원 화면에 동시에 결과 표시

> 여러 계정을 한 컴퓨터에서 동시에 쓰려면 일반 창 + 시크릿 창처럼 **세션이 분리된 창**을 사용하세요.

## 💻 로컬 실행

```bash
# 1) 의존성 설치
npm install

# 2) (선택) LLM 추천을 쓰려면 OpenAI 키 설정
export OPENAI_API_KEY="sk-..."

# 3) 서버 실행
node server.js
```

브라우저에서 접속:

```text
http://127.0.0.1:4173
```

`OPENAI_API_KEY`가 없어도 앱은 정상 실행되며, 장소·메뉴 추천은 서버 내장 규칙 엔진(로컬 추천)으로 동작합니다.

- **Node 버전**: 18 이상이면 어디서나 동작 (네이티브 빌드 의존성 없음)
- 데이터는 `openplanner.json` 한 파일에 저장됩니다. 테스트 계정·일정이 이미 들어 있습니다.

## 핵심 기능

- **회원가입/프로필**: 아이디·비밀번호 가입(bcrypt 해싱), 거주지·선호음식·직장 위치 입력
- **개인 캘린더 + 시간표**: 날짜 클릭 팝업으로 일정 추가/수정/삭제, 카테고리·반복·기간·공개여부, 주간 시간표
- **약속 방(초대코드)**: 주최자가 방 생성, 참여자는 코드로 본인 계정 입장, 서버 기반 실시간 멤버 동기화
- **준비 완료 게이트**: 전원 준비 완료 시에만 주최자가 AI 시간 선정 가능, 결과는 전원에게 동시 표시
- **AI 시간 결정(규칙)**: 일정 충돌 + 컨디션 리스크(전날 야간·다음날 시험) 점수화
- **AI 장소·메뉴 결정(LLM/규칙)**: 선호지역(없으면 평일=직장/주말=거주지) 종합 → 교통 좋은 중간 지점, 회피 반영한 구체적 메뉴

## 보안

- 비밀번호 bcrypt 해싱(12 rounds), 세션 토큰 인증
- 본인 외 비공개 일정 제목은 "바쁨"으로 마스킹, 타인 선호는 서버 응답에서 제외
- XSS 방어(escapeHtml), Path Traversal 방어, 요청 본문 크기 제한

## 기술 스택

- 프론트엔드: HTML / CSS / Vanilla JS
- 백엔드: Node.js (외부 웹 프레임워크 없이 http 모듈)
- 저장소: 순수 JS 파일 저장소(JSON) — 네이티브 빌드 의존성 없이 어디서나 실행
- 인증: bcryptjs (순수 JS 해싱)
- AI: OpenAI API (미설정 시 내장 규칙 엔진 fallback)
